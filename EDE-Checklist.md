# Development Stages Index

Each tool progresses through four stages. Use these definitions to guide when to check off each item:

-   **Built**  
    The tool exists and runs in its basic form. Core functionality is present, but it may be rough or incomplete.

-   **Refined**  
    The tool's UI/UX has been polished. It feels harmonious with the platform, labels are clear, and the flow is intuitive.
v
-   **Tested**  
    The tool has been checked against edge cases and resilience scenarios. It performs reliably under different inputs and conditions.

-   **Verified**  
    The tool works as expected, is selfâ€‘explanatory, and trustworthy. A fresh user can understand its purpose and use it without external notes.
